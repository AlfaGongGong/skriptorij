════════════════════════════════════════════════════════
 main.py
════════════════════════════════════════════════════════
SVRHA: Entry point aplikacije. Pokreće Flask server, otvara browser na Termuxu,
       rukuje signal handlerom za graceful shutdown.

IMPORTI: os, sys, signal, threading, webbrowser, time, logging, pathlib.Path,
         app.create_app, api_fleet.FleetManager, network.model_discovery,
         config.settings, config.logging_config, utils.logging

FUNKCIJE:
  _flask_audit_loop(port)
    → Pozadinska nit koja šalje heartbeat audit log poruke svakih 60 sekundi.
      Prati health stanje aplikacije dok server radi.

  _graceful_shutdown(signum, frame)
    → SIGINT/SIGTERM handler. Zatvara Flask server čisto, bez bacanja traceback-a.

  _resolve_book()
    → Pronalazi putanju do prve EPUB knjige u INPUT_DIR direktoriju.
      Vraća apsolutnu putanju ili None ako nema knjiga.

  _open_browser()
    → Otvara browser na http://localhost:<port>/ uz 1.5s delay.
      Na Termuxu koristi termux-open-url, inače webbrowser.open().

  if __name__ == "__main__" blok
    → Učitava konfiguraciju, inicijalizuje FleetManager, pokreće model discovery,
      startuje Flask server (threaded=True), poziva _open_browser().

 run.py
════════════════════════════════════════════════════════
SVRHA: Alternativni launcher koji implementira inteligentni run loop.
       Određuje strategiju (PREVOD, LEKTURA, RETRO), pametan reset cache-a,
       i koordinira ponovne pokušaje za neuspjele chunkove.

IMPORTI: os, asyncio, json, pathlib, logging, time, shutil,
         core.engine.SkriptorijAllInOne, api_fleet, config.settings,
         processing.retro, processing.pipeline

FUNKCIJE:
  _ucitaj_quality_scores(engine, sharedstats)
    → Učitava quality_scores.json iz checkpoint direktorija i puni shared_stats.

  _odredi_strategiju(engine, html_files, scores)
    → Analizira postojeće checkpointe i ocjene, vraća string strategije:
      "PREVOD" ako nema prijevoda, "LEKTURA" ako treba dorada,
      "RETRO" ako ima loših blokova, "GOTOVO" ako je sve OK.

  _pametni_reset_cachea(engine, scores, sharedstats)
    → Briše samo checkpoint fajlove koji imaju ocjenu ispod praga.
      Čuva dobre checkpointe kako bi se izbjeglo nepotrebno ponavljanje.

  start_skriptorij_from_master(bookpathstr, modelname, sharedstats, shared_controls)
    → Glavna entry funkcija. Inicijalizira engine, odredi strategiju, pokreće
      asyncio event loop s main_loop() korutinom.
    → _retro_pa_nastavi() — inline async; radi RETRO a zatim nastavlja PREVOD.
    → main_loop() — inline async; ponavlja obradu dok nema nedovršenih poglavlja.

════════════════════════════════════════════════════════
 api_fleet.py
════════════════════════════════════════════════════════
SVRHA: Centralni modul za upravljanje API ključevima svih provajdera.
       Implementira FleetManager singleton s per-key rate limitingom,
       cooldown logikom, 429 detekcijom, i live reload bez restarta.

IMPORTI: os, json, time, threading, re, math, asyncio, dataclasses, pathlib,
         logging, typing, network.model_discovery

FUNKCIJE / KLASE:
  _parse_groq_duration(value)
    → Parsira Groq-specifičan "Retry-After" string (npr. "1m30s") u sekunde.

  register_active_fleet(fleet)
    → Registruje FleetManager instancu kao globalni singleton.

  get_active_fleet()
    → Vraća aktivni FleetManager singleton.

  class KeyState:
    → Dataclass koji prati stanje jednog API ključa. Nema cooldown/disable mehanizme.
    __init__(key, provider) — inicijalizira s defaultima
    available() — property; uvijek True (cooldown rješava quota_tracker)
    success_rate — property; calls_ok / (calls_ok + calls_failed)
    masked() — vraća maskirani ključ (zadnjih 6 znakova)
    record_request(provider, key) — bilježi total_requests i last_used
    analyze_response(status_code, headers, body) — broji statuse:
      200→calls_ok, 4xx→calls_rejected[code], 5xx→calls_failed[code]
    to_dict() — serijalizacija u dict za state persistence
    to_ui_dict() — dict za prikaz u UI (bez punog ključa)

  class FleetManager:
    → Singleton klasa koja upravlja svim API ključevima za sve provajdere.
    __init__(config_path, state_path) — učitava ključeve i state
    _load_config() — čita dev_api.json; kreira KeyState; klonira GEMINI→GEMMA
    _resolve_models() — za svaki prov. bira aktivni model iz discovery cache-a
    _save_state() — serijalizuje stanje u api_state.json
    flush_now() — javni poziv za trenutni flush stanja
    reload() — live reload iz dev_api.json bez restarta servera
    get_active_model(provider_upper) — vraća model koji se koristi za provajdera
    get_best_key(provider) — vraća sljedeći dostupni ključ za provajdera
    record_request(provider, key) — bilježi RPM za ključ
    analyze_response(provider, key, status_code, headers, body) — obrađuje HTTP
      odgovor; delegira cooldown na quota_tracker, broji statuse u KeyState
    record_network_failure(provider, key) — bilježi timeout/mrežnu grešku
    toggle_key(provider, key_val) — uključuje ili isključuje API ključ
    revive_all(provider) — resetira cooldown u quota_trackeru za provajdera
    get_fleet_summary() — vraća sažetak stanja flote (ukupno/raspoloživo/greška)
    get_fleet_ui() — vraća kompletne UI podatke za sve provajdere
    get_total_active_keys() — ukupan broj aktivnih ključeva
    load_config(fleet, config_path) — standalone loader za testove

 tts.py
════════════════════════════════════════════════════════
SVRHA: Text-to-Speech modul za generisanje audio fajlova iz prevedenih EPUB knjiga.
       Koristi Kokoro TTS. Procesira checkpointove i generiše MP3/WAV fajlove.

IMPORTI: os, re, pathlib, json, time, logging, html (unescape), bs4 (BeautifulSoup),
         config.settings

FUNKCIJE:
  _html_to_tts_tekst(html)
    → Konvertuje HTML chunk u čisti tekst pogodan za TTS.
      Uklanja tagove, dekodira HTML entitete, normalizuje razmake.

  _generiraj_ttsfilter(html_files, output_path, log_fn)
    → Generiše filtrirani tekst iz svih HTML poglavlja za TTS ulaz.
      Preskače kratka i neprozna poglavlja. Vraća True ako uspije.

  start_from_master(bookpathstr, modelname, sharedstats, shared_controls)
    → Glavna entry funkcija za TTS obradu. Inicijalizira Kokoro model,
      procesira sve chunkove i generiše audio fajlove.
    → log(msg, tip) — inline helper za logging

  _primijeni_checkpointe_na_html(checkpoint_dir, html_files, log_fn)
    → Čita checkpoint JSON-ove i primjenjuje prevedeni tekst na HTML fajlove.
      Koristi se za TTS da bi radio s prevedenim tekstom, ne originalnim.

SVRHA: Ocjenjivanje kvaliteta prevedenih chunkova. Kombinuje heurističku

         api_fleet.get_active_fleet, core.text_utils

FORMULA: composite = heuristika * 0.25 + ai_ocjena * 0.75
         fallback (bez AI) = heuristika * 0.90
         Thresholds: odlično≥8.5, dobro≥6.5, slabo≥4.0, kritično<4.0

    → Uklanja JSON markup iz AI odgovora i vraća čisti tekst ocjene.

  _ekstrahuj_cist_tekst(tekst)

  _izracunaj_heuristicki_score(tekst)
    → Heuristička analiza: provjerava dužinu, udio engleskog, ponavljanja,
      pravilnost rečenica. Vraća (score: float, problemi: list[str]).
      Base score = 9.8, oduzima bodove po nalazima.

  _provjeri_nepromjenjenost(stari_tekst, novi_tekst)
    → Mjeri sličnost starog i novog teksta bigramskom presjekom (0.0-1.0).
      Koristi se da otkrije je li chunk promjenjen u LEKTURA fazi.
    → bigrami(s) — inline helper za generisanje bigramskog skupa.

  _je_nonprose_blok(tekst)
    → Detektuje je li chunk nonprose (zaglavlja, stihovi, tablice, kratki tekst).
      Nonprose blokovi dobivaju lakše kriterije ocjene.

  _scoruj_kvalitetu(engine, chunk_tekst, chunk_idx, file_name, uloga, tip_bloka)
    → Async. Glavna funkcija ocjenjivanja: poziva heuristic score, a zatim

      Sprema rezultat u quality_scores.json i checkpoint.

  _scoruj_batch(engine, chunks)
    → Async. Sekvencijalni batch scorer; poziva _scoruj_kvalitetu() za svaki chunk.

  quality_summary(scores)
