ARHITEKTURA SISTEMA (kratki pregled)
──────────────────────────────────────────────────────────────────────────────
- Backend: Flask (Python) — app.py + api/ + core/ + processing/ + network/
- Frontend: Vanilla JS + HTML/CSS — static/js/ + templates/
- AI: Multi-provider fleet (Gemini, Groq, Mistral, Cerebras, SambaNova, Cohere,
  OpenRouter, GitHub Models, Chutes, HuggingFace, Kluster)
- Checkpointi: per-blok .chk fajlovi u CHECKPOINT_BASE_DIR/_skr_{stem}/checkpoints/
- Izlaz: EPUB fajl u OUTPUT_DIR (Moon+ Reader biblioteka na Androidu)

  Sadrži:

run.py (~328 linija)

processing/pipeline.py (~469 linija)
  Granularni pipeline za obradu jednog bloka (chunka) teksta.
  V2.0 — svaki korak ima vlastiti .chk fajl.
  Sadrži:
  - _chk_path/read/write: helpers za checkpoint fajlove
    (finalni .chk, .prevod.chk, .lektura.chk)
  - obradi_chunk(self, chunk, chunk_idx, file_name, prev_ctx, next_ctx):
    GLAVNA FUNKCIJA — obrađuje jedan blok kroz pipeline korake:
    1. Provjeri checkpoint (resume ako već obrađeno)
    2. PREVOD mod: prijevod → lektor → guardian → validator → quality score
    3. LEKTURA mod: lektor → quality score
    4. Rescue: ako quality < threshold → pokušaj s drugačijim provajderom
    5. Atomično sačuvaj finalni .chk
  - _prijevod_bloka(self, chunk, ...): poziva _call_ai_engine s ulogom PREVODILAC
  - _lektura_bloka(self, sirovo, chunk, ...): poziva _call_ai_engine s ulogom LEKTOR
  - _guardian_provjera(self, chunk, prevod): provjera halucinacija i AI injekcija
  - _validator_provjera(self, chunk, prevod): provjera konzistentnosti s izvorom

processing/workers.py (~350 linija)
  Implementacija specifičnih AI worker koraka koji se pozivaju iz pipelinea.
  Sadrži:
  - _robust_json_parse(raw): parsira JSON iz AI odgovora (5 strategija:
    direktno, bez markdown fences, sve { } blokove, naivno, ast.literal_eval)
  - _call_single_provider(self, provider, model, sys_prompt, user_prompt, ...):
    HTTP poziv jednom specifičnom provideru (koristi http_client)
  - _obradi_guardian(self, chunk, prevod): GUARDIAN prolaz — detekcija
    prompt injekcija, halucinacija, engleski ostatci
  - _obradi_validator(self, chunk, prevod): VALIDATOR prolaz — ocjena
    konzistentnosti prijevoda s izvorom
  - _obradi_polish(self, tekst): POLISH prolaz — finalni stilski sjaj
  - _sanitize_for_epub(tekst): čišćenje za EPUB output (remove AI artefakti)

processing/parallel.py (~125 linija)
  Adaptivni paralelizam za obradu chunka.
  Sadrži:
  - AdaptiveParallelism klasa:
    - _get_window_size(): dinamički računa sliding window (1-6) na osnovu
      broja aktivnih ključeva i RPM limita najsporijeg providera
    - process_chunks_parallel(chunks, file_name, ...): sliding window scheduler
      koji procesira max N chunka istovremeno (backpressure, bez thundering herd)
      s random jitterom između poziva

processing/retro.py (~382 linije)
  Retroaktivna re-lektura — prolaz po već obrađenim .chk fajlovima.
  Sadrži:
  - _je_finalni_chk(path): razlikuje finalne od međukorak .chk fajlova
  - _get_checkpoint_dir(self): locira centralni checkpoint dir
  - retroaktivna_relektura_v10(self, ...): async funkcija koja:
    1. Iterira sve finalne .chk fajlove
    2. Čita quality score za svaki
    3. Blokove ispod thresholda šalje na ponovnu lekturu (LEKTOR → GUARDIAN → POLISH)
    4. Upisuje novi tekst ako je kvalitet poboljšan
  - mark_for_review(checkpoint_dir, scores): piše human_review.json s listom
    blokova koji trebaju manualnu provjeru

processing/rescue.py (~33 linije)
  Hitni rescue za blokove koje pipeline nije mogao spasiti.
  - _spasi_od_sirovog(self, sirovo, chunk, ...): pokušava spasiti loš prijevod
    agresivnom lekturom kroz GEMINI i MISTRAL s većom temperaturom (0.60, 0.75)
